NutriClarte
ENFR Prototype
Barcode scan + condition-aware nutrition guidance

Créez une application inspirée de Yuka, centrée sur le diabète, le cholestérol et l’hypertension.

This prototype shows the core product experience: scan a packaged food, analyze label nutrients, then return simple recommendations personalized to diabetes, LDL cholesterol, and hypertension risk.

Try the demo logic See app architecture

What the MVP does

ScanBarcode, OCR, or search product.
ScoreCondition-specific nutrition risk score.
SwapSafer alternatives in the same category.

Core features

Smart scan intake

Use barcode scan first, then fall back to text search and nutrition label OCR when the product is missing from the catalog.

Condition engine

Score products across added sugar, fiber, saturated fat, sodium, and category context to generate tailored advice for each condition.

Actionable swaps

Recommend lower-sugar, lower-sodium, or lower-saturated-fat alternatives from the same product family instead of generic advice.

Clinical lenses

Diabetes

Prioritize added sugars, total carbs per serving, fiber, serving size inflation, and whether the food is beverage-like or dessert-like.

Cholesterol

Prioritize saturated fat, trans fat if available, fiber, and whole-food composition instead of total fat alone.

High blood pressure

Prioritize sodium density, serving size realism, and processed-food penalties because sodium often accumulates across servings.

Recommendations

Recommended app architecture

Mobile client

Flutter or React Native, camera barcode scanning, nutrition result UI, saved preferences, and multilingual support.

Nutrition data layer

Start with Open Food Facts for broad catalog coverage, then add curated product normalization and your own recommendation metadata.

Rules + personalization

Use a transparent rules engine first, then add machine-learned ranking later so every recommendation remains explainable.

Next build steps

  • Connect a barcode SDK and a product database API.
  • Normalize nutrients to per serving and per 100 g for better cross-product comparisons.
  • Create a clinician-reviewed rule matrix for each condition and food category.
  • Add substitution logic that compares products only within the same category.
  • Store scan history, favorites, and household profiles such as adult with diabetes or low-sodium household.

Important: this prototype is for product design and educational guidance only. It does not diagnose disease or replace personalized medical advice.
